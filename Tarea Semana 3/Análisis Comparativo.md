Análisis Comparativo:

La diferencia principal entre la Programación Tradicional y la Programación Orientada a Objetos (POO) se encuentra en la forma de estructurar y manejar la información. En la tradicional, la atención se dirige principalmente a las funciones o acciones. Los datos como la lista de temperaturas se almacenan en un espacio global, lo que significa que están disponibles para cualquier parte del programa. Aunque esto facilita la lectura y resulta práctico para tareas simples, también implica riesgos: cualquier función podría alterar esa lista sin querer, si quisiéramos procesar dos semanas distintas al mismo tiempo, se produciría conflictos porque todas las funciones trabajarían sobre la misma variable global.

En cambio, la Programación Orientada a Objetos cambia el enfoque al centrarse en entidades y no solo en funciones. Al crear una clase como ClimaSemanal, agrupamos tanto los datos (self.temperaturas) como los métodos (ingresar_temperatura, calcular_promedio) dentro de una misma estructura. Este principio se conoce como encapsulación. En este modelo, las variables dejan de ser globales y pasan a ser propiedad exclusiva de cada instancia creada. Esto protege la información y hace que el programa se comporte de forma más controlada, ya que solo es posible modificar los datos a través de los métodos definidos en la propia clase.

Por último, en cuanto a escalabilidad y reutilización, la POO ofrece ventajas claras. Si el programa necesitara ampliarse para manejar varias semanas o diferentes ciudades, en el modelo tradicional sería necesario duplicar variables y repetir código para evitar mezclar información. Con el enfoque orientado a objetos, basta con crear nuevas instancias de la clase ClimaSemanal, una para cada caso. Cada objeto mantiene su propia lista de temperaturas y trabaja de manera independiente, lo que demuestra por qué la POO se ha convertido en el enfoque preferido para desarrollar software más grande, organizado y profesional.
